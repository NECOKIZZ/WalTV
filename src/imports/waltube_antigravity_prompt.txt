----------------------------------------------------
PROJECT OVERVIEW
----------------------------------------------------
Build a full-stack mobile-first social media web app called WalTube — a platform where AI video creators share their prompts alongside the video results they produced. Think Instagram × GitHub for AI video prompts. Creators post a prompt + video, the community scrolls, copies prompts they like, forks and remixes them, and follows creators whose style they love.

The app is fully responsive and optimized for mobile. It should feel and look like a native iOS/Android app when opened on a phone.

----------------------------------------------------
TECH STACK
----------------------------------------------------
Frontend:     Next.js 14 (App Router) + Tailwind CSS + shadcn/ui
Backend/DB:   Firebase (Firestore + Firebase Auth + Firebase Storage)
AI Auto-Fill: Claude API (claude-sonnet-4-20250514) — called server-side via Next.js API route
Deployment:   Google Cloud Run (via Antigravity native deploy)

  Load both from Google Fonts.

  --indigo:   #6F00FF   (identity color — selections, copy button, active nav, glows)
  Background: #000000 (pure black, dominant)
  Surface:    rgba(255,255,255,0.035) with backdrop-filter: blur(12px)
  Text-1:     #f2f2f2 (primary text)
  Text-2:     #888888 (secondary text)
  Text-3:     #444444 (muted text)
  Indigo glow: rgba(111,0,255,0.32)
  Blue glow:   rgba(24,119,242,0.28)

Border Radius (EVERYTHING must be rounded — no sharp edges anywhere):
  --r-sm:   12px  (thumbnails, icons, small elements)
  --r-md:   18px  (cards, fields, modals inner sections)
  --r-lg:   24px  (prompt cards, collection tiles)
  --r-xl:   32px  (modals, sheets, device shell)
  --r-pill: 999px (ALL buttons, chips, badges, search bars, stats rows, follow pills)

Effects:
  - Ambient indigo radial glow at top of every screen (behind content, z-index: 0)
  - Top-edge light line on every card: 0.5px gradient line from transparent → rgba(111,0,255,0.45) → transparent
  - Bottom nav and top bar: backdrop-filter blur(28px), rgba(0,0,0,0.85) background
  - Cards: glass effect with backdrop-filter blur(12px), rgba(255,255,255,0.035) background
  - Active indigo elements: box-shadow 0 0 16px rgba(111,0,255,0.32)
  - Active blue elements: box-shadow 0 0 16px rgba(24,119,242,0.28)
  - NO sharp corners anywhere in the UI

Responsiveness:
  - Mobile (<480px): Full screen, 100dvh, no shell border radius, edge-to-edge
  - Tablet (481–768px): Centered at 430px max-width
  - Desktop (>768px): Show phone shell centered with screen navigation above it

FIREBASE DATA MODELS
----------------------------------------------------

USERS collection:
  uid, handle, displayName, avatarUrl, bio, links{x, instagram, youtube, website},
  primaryModels: string[] (e.g. ["Sora","Runway","Kling"]),
  followers: number, following: number,
  totalCopies: number, totalPrompts: number,
  createdAt: timestamp

PROMPTS collection:
  id, authorUid, authorHandle, authorAvatar,
  videoUrl (Firebase Storage URL),
  thumbnailUrl (Firebase Storage URL),
  promptText: string,
  model: string (e.g. "Sora", "Runway", "Kling", "Pika", "Hailuo"),
  styleTags: string[] (e.g. ["cinematic","aerial","slowmo"]),
  cameraNotes: string (e.g. "anamorphic, wide angle, slow push-in"),
  moodLabel: string (e.g. "Surreal", "Cinematic", "Abstract"),
  difficulty: string ("Beginner" | "Intermediate" | "Advanced"),
  likes: number, saves: number, copies: number, forks: number,
  isForked: boolean,
  forkedFromId: string | null,
  forkedFromAuthorHandle: string | null,
  createdAt: timestamp

FOLLOWS collection:
  followerId, followingId, createdAt

SAVES collection:
  userId, promptId, collectionName, createdAt

LIKES collection:
  userId, promptId, createdAt

COPIES collection:
  userId, promptId, createdAt

NOTIFICATIONS collection:
  userId, type (follow|like|copy|fork|chain_fork|rating|weekly_digest),
  fromUid, fromHandle, promptId, message, read: boolean, createdAt

AI AUTO-FILL SYSTEM (/api/autofill Next.js route):
  Accepts: { promptText: string, thumbnailBase64?: string }
  Calls Claude API with this system prompt:
  ---
  You are an AI metadata assistant for WalTube, a platform for AI-generated video prompts.
  Given a raw AI video prompt (and optionally a thumbnail image), generate structured metadata.
  Return ONLY valid JSON with no preamble or markdown:
  {
    "styleTags": string[] (3-5 tags from: cinematic, aerial, slowmo, abstract, nature, motion, surreal, urban, macro, timelapse),
    "cameraNotes": string (concise, e.g. "wide angle, slow push-in, warm grade"),
    "moodLabel": string (one of: Cinematic, Surreal, Abstract, Naturalistic, Dramatic, Minimal),
    "difficulty": string (one of: Beginner, Intermediate, Advanced),
    "suggestedModel": string (best guess from: Sora, Runway, Kling, Pika, Hailuo, Other)
  }
  Analyze the complexity and specificity of the prompt to determine difficulty.
  ---
  Populate all Post screen fields with the returned values.

----------------------------------------------------
FOLLOW SYSTEM LOGIC
----------------------------------------------------
- Follow/Unfollow actions update the FOLLOWS collection in Firestore
- Following count on your profile and Followers count on their profile update in real-time
- Feed algorithm: show posts from followed users first (sorted by createdAt desc), then show discovery posts
- When you fork a prompt, auto-follow that creator if not already following
- Suggest creators on onboarding based on selected models

----------------------------------------------------
COPY COUNT TRACKING
----------------------------------------------------
- Every time "Copy" is tapped on any prompt card, increment that prompt's copies count in Firestore
- Also increment the author's totalCopies in their user document
- Log the copy event in the COPIES collection (userId, promptId, createdAt)
- On User Profile, display totalCopies as a prominent stat
- "Copy count" is the primary vanity metric on WalTube — more important than likes

----------------------------------------------------
FORK SYSTEM LOGIC
----------------------------------------------------
- When a user forks a prompt:
  1. A new PROMPTS document is created with isForked: true, forkedFromId, forkedFromAuthorHandle
  2. The original prompt's forks count is incremented
  3. The original author receives a "fork" notification
  4. If the user is not already following the original author, auto-follow them
  5. The forked card shows an attribution banner: "Forked from @originalAuthor"
